# Настройка Claude API

Этот документ описывает подключение Anthropic Claude к проекту `dead-regulations-detector`.

## Когда это нужно

Claude API требуется только для AI-анализа и расширенных объяснений.

Без API-ключа система продолжает работать в режиме rule-based детекции.

## Получение ключа

1. Откройте `https://console.anthropic.com/`
2. Войдите в аккаунт
3. Перейдите в раздел **API Keys**
4. Создайте новый ключ
5. Сохраните значение ключа (обычно начинается с `sk-ant-`)

## Настройка через `.env` (рекомендуется)

```bash
cd ~/dead-regulations-detector
nano .env
```

Укажите параметры:

```bash
AI_PROVIDER=claude
CLAUDE_API_KEY=sk-ant-ваш-ключ
CLAUDE_MODEL=claude-opus-4.5
```

## Настройка через UI

1. Запустите приложение: `http://localhost:8501`
2. В боковой панели выберите `AI Provider = claude`
3. Введите `Claude API Key`

## Проверка подключения

```bash
cd ~/dead-regulations-detector
source venv/bin/activate
python src/nlp/ai_client.py
```

Ожидаемый результат: сообщение об успешном доступе к API выбранного провайдера.

## Что происходит без ключа

При отсутствии `CLAUDE_API_KEY`:

- AI-анализ пропускается
- rule-based детекция продолжает работать
- интерфейс и экспорт остаются доступными

## Частые проблемы

`401 Unauthorized`

- Проверьте корректность ключа
- Убедитесь, что ключ активен и не отозван

`429 Too Many Requests`

- Превышен лимит запросов
- Повторите позже или уменьшите объем анализируемого текста

`413 Payload Too Large`

- Входной текст слишком большой для одного запроса
- Разделите документ на части или используйте более компактный ввод
