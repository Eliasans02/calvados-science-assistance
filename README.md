# Dead Regulations Detector

Проект для **Decentrathon 5.0** (трек AI for Government).  
Система анализирует нормативно-правовые акты Республики Казахстан, выявляет устаревшие и конфликтующие нормы, сравнивает версии документов и формирует объяснимый результат для экспертов и госслужащих.

## О проекте

В законодательных базах накапливаются акты и формулировки, которые формально существуют, но фактически не применимы или создают противоречия.  
Задача проекта — дать инструмент, который помогает быстро находить такие проблемы и снижать ручную нагрузку при правовом анализе.

## Что умеет система

- Анализ загруженных документов (TXT/PDF/DOCX)
- Поиск и анализ документов из `adilet.zan.kz`
- Сравнение двух версий документа с визуальным diff
- Детекция проблем rule-based и через AI (опционально)
- Объяснимый вывод: цитата, объяснение, рекомендация
- Асинхронная статистика и история запусков анализа
- Экспорт результатов (Excel/PDF)

## Основные типы детектируемых проблем

- Устаревшие термины и ссылки на неактуальные органы
- Потенциальные противоречия
- Дублирующие нормы
- Потенциально неприменимые формулировки

## Архитектура

Ключевые модули:

- `src/ui/` — Streamlit-интерфейс и пользовательские сценарии
- `src/analysis/` — движок анализа, оркестрация rule-based и AI
- `src/extraction/` — извлечение и нормализация текста из файлов
- `src/nlp/` — клиенты AI-провайдеров (GitHub/OpenAI/Claude)
- `src/scraper/` — интеграция с поиском и документами Adilet
- `src/metrics/` — сервис статистики и история анализов

Поток обработки:

1. Получение входа (файл/текст/результат поиска)
2. Извлечение и очистка текста
3. Rule-based анализ
4. AI-анализ (если включен)
5. Агрегация результатов и расчет критичности
6. Показ в UI, сохранение статистики, экспорт

## Требования

- Python 3.9+
- pip
- Рекомендуется virtualenv

Для OCR (опционально):

- `tesseract` в PATH
- системные зависимости для `pdf2image`/poppler

## Установка

```bash
git clone https://github.com/Eliasans02/dead-regulations-detector.git
cd dead-regulations-detector
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Конфигурация

Настройки провайдера AI задаются через `.env`:

- `AI_PROVIDER=github|openai|claude|none`
- `GITHUB_TOKEN=...`
- `OPENAI_API_KEY=...`
- `CLAUDE_API_KEY=...`

При `AI_PROVIDER=none` AI-пайплайн полностью отключается.

## Запуск

```bash
source venv/bin/activate
streamlit run src/ui/app.py --server.port=8501
```

Открыть в браузере:

- `http://localhost:8501`

## Поиск по Adilet

Базовый источник — нативная выдача `adilet.zan.kz`.  
В интерфейсе доступны опциональные локальные пост-фильтры (экспериментально):

- локальный фильтр релевантности
- скрытие процессуальных актов

Для поведения “как в старых версиях” эти фильтры нужно оставить выключенными.

## Статистика и история анализов

Система хранит статистику асинхронно (JSON в `data/processed/`) и показывает историю запусков:

- количество обработанных документов
- типы и источники документов
- результаты AI (`success/failed/skipped`)
- результаты анализа (`success/failed`)
- журнал недавних запусков с деталями

## Ограничения

- Качество поиска зависит от индексации и ранжирования на стороне Adilet
- OCR зависит от качества скана и среды выполнения
- Rule-based проверки детерминированы, но не покрывают все юридические кейсы
- Качество AI-вывода зависит от модели и входного контекста
